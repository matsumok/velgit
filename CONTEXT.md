# Velgit — Domain Glossary

## 物件 (Job)
管理の最上位単位。1つの建物または工事に対応する。物件フォルダ内のPDF図面群を内包する。UI上は「Job」と表示する。

## 図面 (Drawing)
1枚のA3サイズPDFファイル。1ファイル1枚。ファイル名はユーザーが管理し、velgitはファイル名を解釈しない。ファイル名・図番・図面名称の命名ルールはユーザーが定める。

## 図面の同一性
ファイル名が安定識別子として機能する。ファイル名が変わった場合はデフォルトで別図面として扱い、履歴は継続しない。ただし、コミット時に引き継ぎ元を手動で指定することで、図面の系譜として履歴を継承できる。

## 図面の系譜 (Drawing Lineage)
ファイル名をまたいで図面の履歴を連鎖させる紐付け。コミット操作の中で、後継図面に対して引き継ぎ元（前のコミットに存在する図面）を1対1で指定する。系譜が設定された場合、履歴表示では旧名称の履歴も連続して参照できる。差分表示も系譜をまたいで比較可能。

## ユーザー (User)
velgitを操作する人。ログイン認証は不要。起動時にユーザー名を入力するだけでよい。コミット・図渡しの操作者として名前をログに記録する。アクセス制限はなく、担当種別の区別もない。

## 初期化 (Initialization)
velgitを既存物件フォルダに導入する操作。既存PDFは取り込まず、導入日以降の変更から管理を開始する。導入時点の図面を「現在の状態」として認識するのみ。

## ワーキングフォルダ (Working Folder)
velgitが監視する物件フォルダ。CADソフトからPDFを書き出す先。変更されたPDFを自動検知してコミット候補としてリストアップする。ドラッグ＆ドロップでもコミット対象に追加できる。UIはWindowsファイルエクスプローラーに近い直感的な操作感を目指す。

## 未コミット (Uncommitted)
ワーキングフォルダ上で変更済みだがまだコミットされていない図面の状態。右ペインの履歴表示においてHEAD選択時に未コミット図面が存在する場合、現在地として「未コミット」ラベルを表示する。

## 差分表示 (Diff View)
2つのコミットまたは図渡し間で、同一図面の変更箇所を視覚的に表示する機能。
両バージョンをラスタライズして画像比較し、変更箇所を赤/青オーバーレイで重ねて1枚の画像として表示する（オーバーレイ差分）。変更図面のリスト表示も併せて提供する。
比較対象：図渡し間・コミット間のどちらも選択可能。

## 図渡し (Drawing Release)
ある時点の図面セットのスナップショットに名前をつける行為。git の Release / Tag に相当。2種類に大別される：
- **社内図渡し**：部署間の情報共有目的
- **社外図渡し**：施主・申請機関・施工会社などへの納品目的

図渡し時に、対象図面を結合した**バインドPDF**（複数図面を1ファイルに結合したPDF）を生成できる。結合順はファイル名の降順。

対象図面の選択：全図面をデフォルトで選択済みとし、不要な図面を外すことで部分選択も可能。どの図面を含めたかは図渡しのメタデータとして記録する。

## リポジトリ (Repository)
物件フォルダをgitリポジトリとして管理する単位。共有ドライブ（Samba）上の物件フォルダを `git init` し、全ユーザーがそのリポジトリに直接アクセスする。ローカルリポジトリは持たない。velgit固有のメタデータは `.git/velgit/velgit.db` に格納する。
