# velgit

PDF図面の変更履歴を、チームで管理・共有するためのデスクトップアプリです。
ファイル名を変えずに誰が・いつ・何を変えたかを記録できます。

---

## velgit でできること

- **変更の自動検知** — フォルダ内のPDFが更新されると自動で検出します
- **コメント付きのコミット** — 「1階平面図 柱位置修正」のようにコメントを添えて変更を保存します
- **変更箇所の視覚確認** — 赤（削除・後退）・青（追加・前進）で差分を重ねて表示します
- **図面セットの作成** — 提出・納品のタイミングで図面一式をまとめ、バインドPDFを生成できます

---

## 基本の考え方

### コミットとは

velgitでは、図面の変更を**コミット**という単位で管理します。

コミットとは「この時点の図面の状態に、コメントを添えてスナップショットを保存する」操作です。  
コミットを積み重ねることで、いつでも過去の版を参照したり、2つの版を並べて比較したりできます。

```
[初回登録] → [コミット1: 1階平面図 作成] → [コミット2: 柱位置修正] → [コミット3: 立面図 追加] → …
```

コミットの情報はフォルダ内の `.git` フォルダに自動で保存されます。  
このフォルダを削除すると履歴がすべて失われます。

---

### 図面一覧の色の意味

中央の図面一覧では、ファイル名の色で変更の状態を示しています。

| 色 | 意味 |
|---|---|
| 緑 | 新規追加されたファイル |
| オレンジ | 意味的変更のファイル（図面の内容が変化している） |
| 水色 | 微小変更のファイル（ファイルは更新されたが図面の内容はほぼ変化なし） |
| 赤（打ち消し線） | 削除されたファイル |
| 色なし | 変更なし |

変更の判定はピクセル比較で行います。2つの版を重ねてピクセルごとにRGBの差を計算し、差が閾値（10/255）を超えるピクセルが1つでもあれば「意味的変更」、閾値以下の差のみであれば「微小変更」と判定します。  
CADツールが図面を再出力した際にアンチエイリアスや内部メタデータだけ変わるケースは、多くの場合「微小変更」として検出されます。

---

### 履歴の引き継ぎ

図番が改訂されてファイル名が変わった場合（例：`101_AA.pdf` → `201_AA.pdf`）、  
そのままでは別のファイルとして扱われ、過去の履歴が参照できなくなります。

コミット時に**引き継ぎ元**を指定することで、旧ファイルの履歴を新ファイルに引き継ぐことができます。  
差分表示でも、引き継ぎ前の版と比較できます。

## 画面の見方

<!-- スクリーンショット: アプリ全体 -->

| エリア | 役割 |
|---|---|
| 左ペイン | プロジェクトの切り替え・コミット履歴（タイムライン）|
| 中央ペイン | 図面の一覧と変更状態 |
| 右ペイン | コミットの作成・図面セットの作成 |

### 左ペイン（タイムライン）の見方

左ペインには、コミットと図面セットが時系列に並んだ**タイムライン**が表示されます。  
上が新しく、下へいくほど過去になります。

```
● 作業中               ← 現在フォルダにあるファイルの状態
│
● 柱位置修正           ← コミット（コメント・作成者・日付）
│
🏷 第1回提出           ← 図面セット
│
● 1階平面図 作成
│
 …（過去）
```

**作業中** はタイムラインの一番上に常に表示されます。  
「今まさにフォルダにあるファイルの状態」を意味しており、まだコミットされていない最新の作業状態です。  
「作業中」を選択すると、最後のコミット以降に変更されたファイルが中央に表示されます。

過去のコミットや図面セットをクリックすると、その時点の図面一覧に切り替わります。

---

## 使い方

### 1. プロジェクトを登録する

「プロジェクト」はvelgitで管理するフォルダの単位です。

1. 左上のプロジェクトセレクターから **「新規プロジェクト登録」** を選択
2. 管理したいPDFが入ったフォルダを選択
3. プロジェクト名を入力して登録

> 複数の工事フォルダを別々のプロジェクトとして登録できます。

---

### 2. 変更をコミットする

PDFを更新すると、中央の図面一覧に変更が自動表示されます。

1. コミットしたいファイルにチェックを入れる
2. 右ペインにコメントを入力（例：「1階平面図 開口寸法修正」）
3. 図番改訂でファイル名が変わった場合は、引き継ぎ元のファイルを指定する
4. **「コミット」** ボタンをクリック

コミット後は左のタイムラインに追加されます。

---

### 3. 過去の版と比べる

1. 左のタイムラインから比較したいコミットをクリック
2. 中央に図面一覧が表示される
3. 図面をクリックすると、差分ビューが開く

変更箇所は赤（削除・後退）・青（追加・前進）のオーバーレイで表示されます。

<!-- スクリーンショット: 差分ビュー -->

---

### 4. 図面セットを作る

提出・納品時に、その時点の図面一式をまとめてスナップショットとして保存できます。

1. 中央ペインでセットに含める図面にチェックを入れる
2. 右ペインで名称・種別（社内／社外）・宛先を入力
3. **「図面セット作成」** をクリック

作成した図面セットからバインドPDF（複数図面を結合した1ファイル）を生成できます。

---

## 注意事項

- 管理対象は **PDFファイルのみ** です
- フォルダ内の `.git` フォルダは削除しないでください（履歴が失われます）
- 同じフォルダを複数のPCで同時に編集する場合は、競合が発生する可能性があります
