# 手動テストチェックリスト

Issues #13–#22 の UI 機能を対象とした手動テスト手順。

---

## 前提：アプリ起動 & フォルダを開く

- [ ] `pnpm tauri dev` でアプリ起動
- [ ] 「ワーキングフォルダを開く」で PDF が入ったフォルダを選択
- [ ] 中央ペインに PDF ファイル一覧が表示される

---

## テスト 1 — 変更種別バッジの表示（#21）

**前提：** PDF を1つ編集するか、新規で追加する

- [ ] CommitPanel（右ペイン上部）に「変更あり」と保留ファイルが表示される
- [ ] 新規ファイル → `M` バッジが付く
- [ ] 変更ファイル（バイト差異あり）→ `M` または `~` バッジが付く
- [ ] 初回コミット前（git 履歴なし）の状態でも保留一覧が出る

---

## テスト 2 — `~` バッジを M に昇格（#22）

**前提：** `~` バッジが付いているファイルがある状態

- [ ] `~` バッジをクリック → `M` に変わる
- [ ] 変わった後の `M` をクリック → 何も変化しない（ダウングレード不可）

---

## テスト 3 — コミット & 保留リストのクリア（#14）

- [ ] CommitPanel のテキストエリアにメッセージを入力して「コミット」ボタンを押す
- [ ] 保留変更リストが消える（パネルごと非表示になる）
- [ ] CommitHistoryPanel に新しい履歴が追加される

---

## テスト 4 — 図面クリック → コミット履歴表示（#18）

- [ ] 中央ペインで図面ファイル名をクリック
- [ ] CommitHistoryPanel にそのファイルのコミット履歴（メッセージ・作者・日時）が表示される
- [ ] まだ一度もコミットしていないファイルをクリック → 「コミット履歴がありません」と表示される

---

## テスト 5 — 作業コピー差分表示（#21）

**前提：** 1回以上コミット済みで、さらに同じファイルを修正した状態

- [ ] CommitPanel でその修正ファイルをクリック
- [ ] CommitPanel 内の DiffView に差分オーバーレイ画像が表示される
  - 変更量が大きい場合：赤（削除）・青（追加）のオーバーレイ
  - 変更がほぼない場合：画像なし

---

## テスト 6 — コミット履歴で2点選択 → 差分表示（#20）

**前提：** 同じファイルに対して2つ以上のコミットがある状態

- [ ] CommitHistoryPanel で古いコミット（旧版）をクリック → ハイライトされる
- [ ] 別のコミット（新版）をクリック → DiffView に差分オーバーレイが表示される
- [ ] 「変更なし」「微小変更」「意味的変更」のいずれかのラベルが表示される
- [ ] 同じコミットを2回選択した場合 → 「変更なし」になる

---

## テスト 7 — ファイル変更の自動検知（#15/#16）

- [ ] ワーキングフォルダを開いた状態で、アプリの外から PDF を追加または更新
- [ ] CommitPanel が自動更新され、保留ファイルとして表示される（手動リロード不要）
- [ ] 画面下部にポーリングモードのバナーが表示されている場合はネイティブ検知が無効（正常）
