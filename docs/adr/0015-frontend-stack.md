# フロントエンドスタックの確定

velgitのフロントエンド技術選択をまとめて記録する。

## 採用技術一覧

| 用途 | 採用技術 | 備考 |
|---|---|---|
| UIフレームワーク | React + TypeScript | Tauri v2標準 |
| スタイリング | Tailwind CSS v4 + `@tailwindcss/vite` | 設定ファイル不要 |
| コンポーネント | shadcn/ui（Lyraプリセット） | Radix UI基盤 |
| アイコン | Lyraプリセットデフォルト（lucide-react） | |
| 非同期状態管理 | TanStack Query | `invoke()`をクエリとして扱う |
| グローバル状態 | Zustand | ファイル変更イベント受信・物件選択状態 |
| フォーム | React Hook Form | Zodなし（バリデーションはRust側） |
| ファイルD&D | react-dropzone | ファイルシステム→アプリのドロップのみ |
| 日時フォーマット | ネイティブDate API（ISO表示） | ライブラリ不要 |
| ルーティング | なし | 物件切り替えはZustand状態で管理 |
| テスト | なし（MVP完成後に検討） | |

## UIレイアウト

3ペイン構成：
- **左ペイン**：コミット履歴ツリー（最新が上、日時＋メッセージ）
- **中央ペイン**：全図面リスト（ファイル名降順）。前回コミットとの差分状態をVSCode/Zed風に色で表示。比較対象コミットは変更可能
- **右ペイン**：デフォルトは差分表示（変更なしはプレビュー）。コミットモードに切り替え可能

**トップバー**：物件セレクタ（ドロップダウン）＋担当種別セレクタ

## PDF表示

- ダブルクリック → システムPDFビューア（`shell.open()`）
- アプリ内プレビュー・差分 → pdfium-render → 一時ファイル → Tauriカスタムプロトコル（ADR-0010と同一パイプライン）
