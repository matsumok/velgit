# 3ペインレイアウトの再設計

## 決定

UIを以下の3ペイン構成に再設計する。

### 左ペイン — タイムライン

- 上部：物件名 + ユーザー名（コンパクト表示）
- 下部：プロジェクト全体のコミット + 図渡しを統合した縦線タイムライン
  - HEAD を最上部に配置
  - コミット：丸点
  - 図渡し：タグバッジ

左ペインで「時点」を選択することが、アプリ全体の表示状態を決定する。

### 中央ペイン — 図面一覧

- 選択された時点（コミット or HEAD）の図面一覧を表示
- 下部アクションエリア：
  - HEAD 選択 + 未コミット変更あり → コミットボタン
  - HEAD 選択 + 未コミット変更なし（HEAD == 最新コミット）→ 図渡しボタン
  - 過去コミット選択 → 何も表示しない（閲覧専用）

### 右ペイン — 図面詳細

- 上部：選択図面のコミット履歴リスト（シンプルなテキストリスト）
  - その図面を変更したコミット：通常表示
  - その図面を変更していないコミット：グレーアウト
- 下部：差分 / プレビューエリア
  - デフォルト：選択時点のファイルプレビュー
  - 過去コミットを選択した場合：その2点間の差分表示

## 背景

従来の実装では右ペインに CommitPanel・CommitHistoryPanel・ReleasePanel・ReleaseHistoryPanel が縦積みされており、情報過多だった。また左ペインは物件名とユーザー名のみで活用されていなかった。

## トレードオフ

**採用した理由：**
- 「時点を選ぶ → 図面を見る」という操作順序がより自然
- 図渡しとコミットを同一タイムラインで管理できる
- 右ペインをプレビュー・差分に集中させることで視線の優先度が明確になる

**代替案：**
- 右ペインをタブ切り替え（差分/履歴/図渡し）にする案 → 却下。タブよりタイムライン起点の操作フローのほうが直感的。
- コミット履歴を右ペインのタブにする案 → 却下。左ペインのタイムライン選択と右ペインの図面履歴を2階層にする現案のほうが整理されている。

## 実装上の影響

### 新規バックエンドコマンドが必要
- `get_project_commits()` — プロジェクト全体のコミット一覧（HEAD 含む）
- `get_drawings_at_commit(oid)` — 指定コミット時点の図面一覧

### 採用する shadcn/ui コンポーネント
- `ScrollArea` — 左・右ペインのスクロール
- `Button` — コミット・図渡し・各アクション
- `Dialog` — 初回ユーザー名入力モーダル
- `Badge` — 図渡しのタグ表示

### 廃止・移動するコンポーネント
- `ReleasePanel` — 中央ペイン下部の図渡しボタンに統合
- `ReleaseHistoryPanel` — 左ペインのタイムラインに統合
- `CommitHistoryPanel` — 右ペイン上部のリストに再実装
- `ThreePaneLayout` — レイアウト幅の変更に合わせて再実装
