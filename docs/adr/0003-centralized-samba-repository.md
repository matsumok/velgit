# Sambaを中央リポジトリとして直接アクセスする（ローカルリポジトリなし）

gitのような分散型（各PCにローカルリポジトリ＋push/pull）ではなく、Samba共有ドライブ上の `.velgit/` フォルダを中央リポジトリとして、各ユーザーのvelgitが直接読み書きする集中型アーキテクチャを採用する。

設計事務所では「共有ドライブ＝全員が見る場所」という運用が定着しており、ローカルリポジトリのpush/pullという概念は不要な複雑さを生む。また全員が同じ履歴を即座に参照できることが重要。ネットワーク常時接続が前提だが、社内Sambaであればオフライン作業は想定しない。

## Considered Options

- **分散型（gitライク）**：各PCにローカルリポジトリを持ちpush/pull。オフライン対応できるが、push忘れや競合解決の操作が発生する。
- **集中型・直接アクセス（採用）**：操作がシンプルで、全員が常に最新の履歴を参照できる。Samba常時接続が前提。
