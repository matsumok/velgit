# 微小変更の検出・スナップショット保存・手動オーバーライド

## 背景

RevitやAutoCADは全図面を一括でPDF出力することが多く、内容に変更がない図面も毎回書き出される。このとき、レンダリングのアンチエイリアスや内部メタデータの違いにより、バイナリ（ハッシュ）は異なるがピクセル差分は閾値以下になることがある。

## 変更状態の3分類

| 状態 | 判定条件 | UIインジケーター |
|---|---|---|
| **変更なし** | ハッシュ同一 | 表示なし |
| **微小変更** | ハッシュ異なる・ピクセル差分が閾値以下 | `~`（薄い色） |
| **意味的変更** | ハッシュ異なる・ピクセル差分が閾値超え | `M`（通常色） |

## ストレージ戦略

ハッシュが異なるファイルはすべて `objects/{hash}/` にスナップショットとして保存する（コミット時点の完全な状態を保証するため）。ハッシュが同一のファイルは既存オブジェクトを参照し、新規保存しない。

コミットには「意味的変更」「微小変更」の両方が自動的に含まれる。「変更なし」（ハッシュ同一）のファイルはコミットに記録されない。

## 手動オーバーライド

精度の問題で「微小変更」と判定されたが実際には意味的変更である場合、ユーザーがステータスを手動で変更できる。

- **コミット前**（主）：コミットUIで各ファイルのステータスを `微小変更 → 意味的変更` に変更してからコミット
- **コミット後**（補）：過去のコミット記録のステータスを遡って修正可能。オーバーライドされた記録にはフラグを立て、手動修正であることを明示する

## SQLiteスキーマへの影響

`commit_files` テーブルに以下を追加：
- `change_type`: `NONE` / `MINOR` / `MEANINGFUL`
- `change_type_manually_overridden`: bool
