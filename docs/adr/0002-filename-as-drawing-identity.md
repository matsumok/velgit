# ファイル名を図面の安定識別子とする

図面のバージョン追跡において、UUIDなどのシステム生成IDではなくファイル名（例：`S-001_1階伏図.pdf`）を安定識別子として使用する。ファイル名が変わった場合は別図面として扱い、履歴は継続しない。

実務では図番や図面名称が変更されることがあるが、その際に履歴が途切れることをユーザーが許容している。UUID方式にすると「ファイル名が変わっても同一図面と判定する」ための手動紐付けUIが必要になり、操作の複雑さに対してメリットが小さい。

## Consequences

図番変更（例：`S-101` → `S-201`）や図面名称の変更時に、過去の履歴と新ファイルは別図面として管理される。意図的な割り切り。

## Amendment

ADR-0026 により部分改訂。コミット時に引き継ぎ元を手動指定することで、ファイル名をまたいだ履歴継承がオプションとして可能になった。基本方針（ファイル名が安定識別子）は維持。
