# スナップショットダウンロードペインの統合設計

## 決定

中央ペイン下部のダウンロードエリアを、過去コミット（browse）と図渡し（release）で共通の `SnapshotDetailPanel` コンポーネントに統合する。

### 表示ルール

| 要素 | 過去コミット | 図渡し |
|------|------------|-------|
| 種別バッジ（社内/社外） | 非表示 | 表示 |
| 図渡し名 | 非表示 | 表示 |
| 相手先（recipient） | 非表示 | 表示（あれば） |
| GitCommit アイコン + OID + メッセージ | 表示 | 表示 |

### チェックボックスとダウンロード対象

| モード | チェックボックス | ダウンロード対象 |
|--------|---------------|----------------|
| 過去コミット（browse） | 全行に追加・全選択デフォルト | チェックされた図面 |
| 図渡し（release） | なし | 図渡し登録時の全図面 |

図渡しは作成時に対象図面が確定したスナップショットであるため、再選択は不要。

### ボタン配置

`[ArchiveIcon + ".zip"] [DownloadSimpleIcon + "一式PDF"]`

### ファイル命名

`YYMMDD_ラベル.pdf` / `YYMMDD_ラベル.zip`（一式 PDF と同パターン、拡張子のみ変える）

## バックエンドコマンド

| コマンド | 変更区分 | シグネチャ |
|---------|---------|----------|
| `generate_bind_pdf` | 既存・変更なし | `(release_id, save_path)` |
| `generate_release_zip` | 新設 | `(release_id, save_path)` |
| `generate_commit_bind_pdf` | 引数追加 | `(commit_oid, filenames[], save_path)` |
| `generate_commit_zip` | 新設 | `(commit_oid, filenames[], save_path)` |

ZIP 生成には `zip` クレート（v2）を使用する。

## 背景

- 過去コミットと図渡しで別々のコンポーネント（`CommitDetailPanel` / `ReleaseDetailPanel`）が存在し、ボタン表記・スタイルが乖離していた
- 過去コミット閲覧時にも図面の選択ダウンロードが必要という要件が追加された
- バラ PDF を ZIP でまとめてダウンロードする機能を両コンテキストで提供したい

## トレードオフ

**採用した理由：**
- browse / release の両モードで「時点のスナップショットをダウンロードする」という操作は同一概念であり、共通コンポーネントが自然
- 図渡しはチェックボックスなし（登録時の全図面）とすることで、図渡しの意図（意図的に選んだセット）を保護する

**代替案：**
- 図渡しにもチェックボックスを付ける案 → 却下。図渡し作成時に対象図面を決定済みのため、事後に変えるのは意味が薄い。
- コマンドを filenames[] 統一シグネチャにする案 → 却下。図渡し系はリリース ID で DB から取得するほうが変更に強く、フロントエンドで drawing リストを取り回す必要がない。
