# 右ペインへの差分モードトグル追加

ADR 0019 で定めた右ペインの挙動を更新する。

## 決定

右ペイン最上部に「差分表示」トグルを追加し、プレビューモードと差分モードを切り替え可能にする。

### off mode（プレビューモード）

- 「現在」セクションおよび履歴コミットがすべてクリッカブルなアイテムになる
- クリックしたアイテムのバージョンのプレビューを下部に表示する
- 「現在」クリック → 現在（HEAD または作業コピー）のプレビュー
- 履歴コミットクリック → そのコミット時点のプレビュー
- 選択解除はなし。アイテムを切り替えることで表示が変わる

### on mode（差分モード）

- ADR 0019 の差分表示ロジックを維持（選択履歴コミット vs 現在の差分）
- 「現在」アイテム：履歴コミット選択中は青ハイライト（diff overlay の青と対応）
- 選択中の履歴コミット：赤ハイライト（diff overlay の赤と対応）
- トグル ON 切り替え時：変更履歴の最新コミットを自動選択

### トグルの状態管理

- グローバル状態（Zustand store）で保持。アプリ再起動でリセット
- デフォルト: off
- ユーザーのクリック操作のみでトグル状態を変える
- 変更履歴がない図面を選択した場合：トグルは ON のままグレーアウト・disabled になる。図面を切り替えても diff mode の意図を保持し、履歴のある図面に戻ったとき即座に diff mode が有効になる

## 背景

ADR 0019 の右ペインは「過去コミット選択 → 差分表示」のみだった。ユーザーが「履歴の任意のバージョンをそのまま確認したい（diff ではなく元画像を見たい）」というユースケースが生まれたため、プレビューと差分を切り替えられるモードを追加した。

## トレードオフ

**採用した理由：**
- diff overlay の色（赤/青）とリストのハイライト色を統一することで、どのコミットが「現在」でどれが「比較元」かを一目で判断できる
- 履歴のない図面でもトグル状態を保持することで、複数ファイルを横断的に確認するワークフローでの意図的な操作を保護できる

**代替案：**
- 「履歴なし図面に切り替えたら強制 off にする」案 → 採用せず。履歴なしファイルを誤って選んでも diff mode を継続したいケースがあるため
- タブで「プレビュー/差分」を切り替える案 → 採用せず。ヘッダー領域を小さく抑え、コンテキストを維持したままモード切り替えできるトグルのほうがコンパクト
