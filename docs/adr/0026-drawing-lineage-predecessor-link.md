# 図面の系譜（Drawing Lineage）によるファイル名変更時の履歴継承

ADR-0002 を部分的に改訂する。ファイル名が安定識別子であるという基本方針は維持しつつ、コミット時の手動指定により改名をまたいだ履歴継承を可能にする。

## Context

ADR-0002 は「ファイル名が変わった場合は別図面として扱い、履歴は継続しない」と定めた。しかし実務では図番改訂（例：`101_AA.pdf` → `201_AA.pdf`）や枝番付与（例：`101_AA.pdf` → `101_AA(1).pdf`）が頻繁に発生し、内容は同一のまま名称だけが変わるケースで履歴が断絶することが課題となった。

## Decision

`drawing_lineage` テーブル（SQLite）を追加し、後継ファイル名と引き継ぎ元ファイル名を 1:1 で紐付ける。紐付けはコミット操作の中でのみ設定可能。前のコミットに存在するファイルを引き継ぎ元として手動で指定する。

- 設定しない場合の動作は ADR-0002 のまま（別図面として独立）
- 1対1のみ対応（分割・統合は対象外）
- 誤設定の修正 UI は将来対応（Phase 2）

履歴取得（`lineage_history`）は系譜チェーンを再帰的に辿り、各ファイル名の git log を結合してタイムスタンプ順に返す。差分表示も系譜をまたいで blob を解決する。

## Consequences

- ファイル名変更時に意図的に履歴を継承させることができる
- 引き継ぎ元を指定しなければ従来通り独立した図面として扱われる
- 系譜は前のコミット（git HEAD）のファイルからのみ選択可能であり、時間は常に一方向のため循環は発生しない
- 既存の `velgit.db` は `drawing_lineage` テーブルが空のまま追加されるだけで、動作は従来と同一
