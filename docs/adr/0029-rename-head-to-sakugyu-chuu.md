# タイムラインの「HEAD」表記を「作業中」に変更する

## Context

タイムライン最上部の項目は「HEAD（作業コピー）」と表示していた。

この「HEAD」はgitの用語だが、velgitの文脈では意味がずれている。

- **gitにおけるHEAD** = 最後にコミットされた状態を指すポインタ
- **velgitの「HEAD」が指すもの** = 今フォルダにある生のファイルの状態（gitでいうworking tree）

velgitのユーザーはgitを知らない前提であり、「HEAD」は無意味なjargonになる。
さらにgitを知っているユーザーにとっては誤った意味に読める。

## Decision

タイムラインの当該項目のラベルを **「作業中」** に変更する。

- 「過去のコミット」に対して「今まさに作業している状態」という対比が直感的に伝わる
- gitの用語と混同されない
- 「作業コピー」のサブラベルも不要になるため削除する

## Consequences

- `ProjectTimeline.tsx` の `HeadItem` ラベルを変更
- README・CONTEXT.md 内の関連記述を合わせて更新
- git用語の「HEAD」はコード・DB・API層では引き続き使用する（内部識別子としては正確なため）
