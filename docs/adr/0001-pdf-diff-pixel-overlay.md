# PDF差分はピクセルオーバーレイで行う

建築図面のPDF差分表示として、ファイルレベルの変更リストやサイドバイサイド比較ではなく、両バージョンをラスタライズして画像比較し、変更箇所を赤/青オーバーレイで重ねる方式を採用する。設計スタッフが「どこが変わったか」を一目で把握することが最大の価値であり、ファイルリストでは変更箇所の特定に手間がかかりすぎるため。

## Considered Options

- **ファイルリスト**：どの図面が変わったかを列挙するのみ。実装は最も簡単だが、変更箇所を見つけるために図面を自分で見比べる必要がある。
- **サイドバイサイド**：旧版と新版を左右に並べて表示。変更箇所はわかるが、大きな図面では見比べに手間がかかる。
- **ピクセルオーバーレイ（採用）**：変更箇所が1枚の画像で明示される。実装コストは最も高いが、建築図面における実務的な価値が最も高い。

## Consequences

pdfium（またはそれに相当するPDFレンダラー）への依存が発生する。A3図面を十分な解像度（150〜300 DPI）でラスタライズする処理コストを考慮する必要がある。
